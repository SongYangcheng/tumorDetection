# 架构设计

## 系统架构

tumorDetection采用前后端分离的微服务架构设计。

```
┌─────────────────┐    HTTP/REST    ┌─────────────────┐
│   Vue.js前端     │◄──────────────►│   Flask后端      │
│                 │                 │                 │
│ - 用户界面      │                 │ - API服务       │
│ - 图像上传      │                 │ - 模型推理      │
│ - 结果显示      │                 │ - 数据处理      │
└─────────────────┘                 └─────────────────┘
         │                                   │
         ▼                                   ▼
┌─────────────────┐                 ┌─────────────────┐
│   浏览器客户端   │                 │   YOLO11模型     │
└─────────────────┘                 └─────────────────┘
```

## 组件说明

### 前端组件

#### 视图层 (Views)
- **HomeView**: 应用首页，介绍和导航
- **UploadView**: 图像上传界面
- **ResultsView**: 检测结果展示

#### 服务层 (Services)
- **api.ts**: 后端API调用封装

#### 状态管理 (Stores)
- **counter.ts**: 示例状态管理（可扩展）

### 后端组件

#### 应用层 (main.py)
- Flask应用实例
- 路由定义
- CORS配置

#### 模型层 (models/)
- YOLO11模型权重文件
- 模型配置文件

#### 工具层 (utils/)
- **image_processing.py**: 图像预处理和后处理

## 数据流

1. **图像上传**
   ```
   用户 → 前端表单 → API调用 → 后端接收 → 图像验证 → 返回确认
   ```

2. **肿瘤检测**
   ```
   图像数据 → 预处理 → YOLO模型 → 后处理 → 检测结果 → 前端展示
   ```

## 技术决策

### 为什么选择Flask？
- 轻量级，易于学习
- 丰富的生态系统
- 适合中小型API服务

### 为什么选择Vue.js？
- 组件化开发
- 响应式数据绑定
- 活跃的社区支持

### 为什么选择YOLO11？
- 最新的目标检测算法
- 优秀的性能和准确率
- 支持实时推理

## 扩展性考虑

### 水平扩展
- 后端可部署多个实例
- 使用负载均衡器分发请求
- 模型推理可使用GPU集群

### 垂直扩展
- 添加更多AI模型
- 支持多种图像格式
- 集成医疗数据库

### 安全性
- API认证和授权
- 数据加密传输
- 隐私保护合规
